## 1. Введение

Zvooq — учебный музыкальный веб‑сервис для коллекционирования, организации и прослушивания музыки. Проект предназначен для практики работы с DOM, асинхронным JavaScript, файловой загрузкой, простым REST API и клиентским хранением состояния.

Приложение сочетает многостраничную структуру и SPA‑поведение: страницы доступны напрямую, но при навигации используется подмена `main`, что сохраняет работу глобального плеера.

## 2. Цели и назначение продукта

- Обучение разработке web‑приложений среднего масштаба.
- Демонстрация полного цикла: интерфейс, API, хранение данных, обработка файлов.
- Реализация полноценной музыкальной библиотеки с избранным, плейлистами, статистикой и архивом.

## 3. Ключевые возможности

---

## 4. Архитектура проекта

Проект сочетает MPA и SPA‑подход:

- **MPA**: каждая страница имеет собственный HTML.
- **SPA**: `js/spa.js` перехватывает навигацию и подменяет `main` без перезагрузки страницы.

Бэкенд построен на **Node.js + Express**. Данные хранятся в JSON‑файлах внутри репозитория.

## 7. Публичный доступ через ngrok

Для демонстрации проекта из внешней сети:

1. Запустите сервер:

```bash
npm run start
```

2. В отдельном терминале запустите туннель:

```bash
npm run tunnel
```

3. Получите публичный URL вида:

```
https://xxxxxxxx.ngrok-free.app
```

Установка ngrok (рекомендуется официальный агент v3+):

```bash
brew install ngrok/ngrok/ngrok
```

Или скачайте установщик с официального сайта ngrok и установите вручную.

Если ngrok требует токен:

```bash
ngrok config add-authtoken <ВАШ_ТОКЕН>
```

## 10. API: обзор

- `GET /api/tracks`
- `POST /api/tracks`
- `PUT /api/tracks/:id`
- `DELETE /api/tracks/:id`
- `POST /api/tracks/:id/favorite`

### Альбомы
- `GET /api/albums`
- `PUT /api/albums/:id`
- `DELETE /api/albums/:id`

### Плейлисты
- `GET /api/playlists`
- `POST /api/playlists`
- `PUT /api/playlists/:id`
- `DELETE /api/playlists/:id`
- `POST /api/playlists/:id/tracks`
- `DELETE /api/playlists/:id/tracks/:trackId`

### Авторизация
- `POST /api/auth/login`
- `POST /api/auth/register`

## 11. API: подробная спецификация

### 11.1 GET /api/tracks

**Описание:** вернуть список всех треков.

**Ответ:**

```json
[
  { "id": 1, "title": "Demo", "artist": "Artist", "duration": 180 }
]
```

### 11.2 POST /api/tracks

**Описание:** загрузка трека или альбома.

**Формат:** `multipart/form-data`.

**Ключи:**
- `audio`, `cover`, `title` или `albumTitle`, `artist`, `genre`, `description`, `label`, `releaseDate`, `userId`.
- Для альбомов: `trackTitles`, `trackArtists`, `trackGenres`, `trackDescriptions`, `durations`.

**Ответ (трек):**

```json
{ "success": true, "type": "track", "data": { "id": 1 } }
```

**Ответ (альбом):**

```json
{ "success": true, "type": "album", "data": { "id": 10 }, "tracks": [] }
```

### 11.3 POST /api/playlists/:id/tracks

**Описание:** добавить трек в плейлист.

```json
{ "trackId": 1, "userId": 123 }
```

## 12. Клиентская часть: SPA и маршрутизация

SPA‑логика реализована в `js/spa.js` и включает:

- перехват ссылок;
- `fetch` HTML;
- замену `<main>`;
- вызов `initX()`;
- fade‑анимацию.

---

## 13. Клиентская часть: общие утилиты

Файл `js/shared.js` содержит:

- API‑обёртки (`API.get/post/put/delete/postForm`).
- Авторизация (`Auth.getUser/setUser/logout`).
- Уведомления (`showToast`).
- Форматирование (`formatDuration`, `formatDate`).
- Глобальный плеер (`Player`).
- Рендер шапки (`renderHeader`).
- Логи (`ZLog/ZError`).

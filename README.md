# Проект: Адаптивный сайт для строительных компаний

## Введение
Кратко: цель проекта — разработать адаптивный сайт для строительной компании с удобным UI, мобильной версией и базовым функционалом представления услуг, портфолио, контактов.

---

## 1. Теоретическая часть

### 1.1 Анализ предметной области адаптивного сайта для строительной компании
- Объект: строительная компания (услуги, проекты, отзывы, контакты).
- Требования: доверие, простота, адаптивность, SEO.
- Аудитория: заказчики, партнёры, инвесторы.

### 1.2 Проектирование будущего интерфейса для адаптивного сайта для строительной компании
- Структура страниц: главная, услуги, проекты, о компании, контакты.
- Мобильные/планшет/десктоп.
- Элементы: хедер, навигация, карточки услуг, галерея, CTA, формы.

### 1.3 Выбор технологий для построения адаптивного сайта строительной компании
- HTML5, CSS3, JS.
- Адаптив: медиа-запросы (`@media`), Flexbox/Grid.
- Библиотеки (по желанию): Normalize, Swiper, иконки (Material/FontAwesome).

## 2. Практическая часть: Создание адаптивного сайта для строительных компаний

### 2.1 Реализация HTML-разметки сайта
- Семантическая структура (`<header>`, `<main>`, `<section>`, `<footer>`).
- Кроссбраузерные атрибуты, доступность (alt, aria-).
- Пример: секция "Услуги", секция "Проекты".

### 2.2 Реализация CSS-стилей сайта
- Базовые переменные, палитра, типографика.
- Сетка: `display: grid` / `flex`.
- Адаптивные брейкпоинты: 320/768/1024/1440.
- Переходы, hover, состояния.

### 2.3 Реализация JavaScript-функционала сайта
- Бургер-меню для мобильных.
- Слайдер/фильтр проектов.
- Скролл к секции.
- Валидация формы обратной связи.

### 2.4 Тестирование сайта на соответствие поставленным требованиям и отработки функционала
- Проверка UI на разрешениях.
- Кроссбраузерный тест (Chrome, Safari, Firefox).
- Функциональный: отправка формы, открытие меню, адаптивность.
- Соответствие требованиям: все секции доступны, логика работает.

## Заключение
Проект реализован как адаптивный сайт для строительной компании. Поддерживает основные сценарии: представление услуг, проектов, связь с клиентом. Удовлетворяет требованиям мобильности и UX.
